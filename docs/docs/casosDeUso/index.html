Casos de uso

Caso de uso 1 (Criar usuário)​

  • Identificador 1
  • Nome: Criar usuário
  • Resumo: Esse caso de uso descreve as etapas para um usuário se cadastrar
  • Ator: Usuário
  • Prioridade: Essencial
  • Requisitos não funcionais associados: O sistema deve ser fácil de usar, e o sistema deve ter conexão com o MySqL
  • Pré-condições: possuir um email
  • Pós-condições: O usuário estar cadastrado no sistema
  • fluxo de eventos principal:
    1. Começa quando o usuário abre o app
    2. O usuário clica em se cadastrar
    3. O usuário insere seu email, data de nascimento, e escolhe uma senha
    4. O usuário submete seus dados ao sistema
    5. O sistema cadastra os dados do usuário
  • fluxos secundários:
    1. caso o usuário digitar uma data de nascimento ou e mail inválidos, o sistema retorna um erro

Caso de uso 2 (Fazer Login)​

  • Identificador: 2
  • Nome: Fazer Login
  • Resumo: Esse caso de uso descreve as etapas para um usuário realizar o login na plataforma
  • Ator: Usuário
  • Prioridade: Essencial
  • Requisitos não funcionais associados: O sistema deve ser fácil de usar, o sistema deve ter conexão com o MySQL
  • Pré-condições: O usuário estar cadastrado no sistema
  • Pós-condições: O usuário estar logado no sistema
  • fluxo de eventos principal:
    1. Começa quando o usuário abre o app
    2. O usuário insere seu email e senha
    3. O usuário submete seus dados ao sistema
    4. O sistema verifica os dados, e faz a autenticação do usuário
  • fluxos secundários:
    1. O usuário apertar em “esqueci minha senha” caso não consiga realizar o login
    2. Caso o usuário digitar credenciais incorretas, o sistema retornará um erro

Caso de uso 7 (Criar evento)​

  • Identificador: 7
  • Nome: Criar evento
  • Resumo: Esse caso de uso descreve as etapas para um usuário cadastrar um evento
  • Ator: Usuário
  • Prioridade: Essencial
  • Requisitos não funcionais associados: O sistema deve ser fácil de usar, o sistema deve ter conexão com o MySQL
  • Pré-condições: O usuário estar logado no sistema
  • Pós-condições: O usuário ter seu evento cadastrado no sistema
  • fluxo de eventos principal:
    1. Abra o app, selecione o botão de “+”
    2. Na tela de cadastro, digite um nome, uma hora pra começar, uma data e uma descrição do evento.
    3. selecione se quer ser notificado do evento
    4. aperta em salvar
  • fluxos secundários:
    1. Selecione uma data, e aperte em adicionar
    2. Siga a partir do passo 2 acima
